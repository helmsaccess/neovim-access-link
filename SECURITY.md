# Security policy

## Supported versions

The project is currently alpha-to-beta software. Security fixes target the
latest published test build and the current default branch; older development
builds are not maintained.

## Reporting a vulnerability

Prefer GitHub private vulnerability reporting or a private repository security
advisory when available. Otherwise, contact Emanuel Helms privately at
`emanuel@helmsaccess.de`. Do not place exploit details, credentials, hostnames,
editor text, or private paths in a public issue.

Include affected version, platform, impact, reproduction prerequisites, and a
redacted diagnostic report. Do not test against systems or accounts you do not
own or have explicit permission to use.
