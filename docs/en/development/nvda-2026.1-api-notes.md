# NVDA 2026.1.1 API review

Reviewed source: official tag `release-2026.1.1`, commit
`5d92106f17e461dac62aa48257bbbf4183e033d0`.

Confirmed points include 64-bit Python 3.13.12, GlobalPlugin/AppModule script
and event behavior, event-queue handoff through `queueHandler`, diff-based and
UIA-notification Windows Terminal paths, typed-character handling, validated
manifest API versions, settings category registration, and configuration
profile callbacks.

The official developer guide documents `inputCore.decide_handleRawKey` as a
public `Decider`, but it is too early for this observer: returning `True` only
permits further NVDA processing. The AppModule instead observes F12 through
the public `inputCore.decide_executeGesture` decider and returns `True` without
binding a script. NVDA then raises `NoInputGestureAction`, causing the keyboard
hook to pass the original physical event to the OS. Claim work is queued
separately and never runs on the input-hook thread. Only after an F12 match does
the decider query `api.getFocusObject()`; it accepts only that concrete,
still-registered AppModule instance and the same complete terminal identity in
the gate.
Neovim's corresponding `vim.on_key` observer likewise schedules its session-file
write with `vim.schedule()` rather than invoking regular Vim functions from
the key callback. Neovim 0.11 adds the public return contract which lets a
callback consume a key by returning `""`; Neovim 0.10 does not yet have that
contract.

The official English UI strings confirm these paths:

- `NVDA menu → Preferences → Settings...`
- `NVDA menu → Preferences → Input gestures...`
- `NVDA menu → Tools`
- `NVDA menu → Configuration profiles...`

Suppression may skip downstream handling only for an active, authenticated,
focused, exactly bound Neovim session. Timeout, exception, focus loss,
termination, or deactivation must restore normal handling.

Windows Terminal controls across tabs/windows and scripts were tested with NVDA 2026.1.1;
other front ends and broader NVDA/UIA versions remain unapproved.
